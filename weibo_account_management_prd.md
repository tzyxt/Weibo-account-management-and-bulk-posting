---

## 2. 产品背景

微博运营场景中，用户可能需要同时管理多个微博账号。传统方式需要频繁切换浏览器、切换账号、重复登录、复制内容并逐个发布，效率较低，也不方便统一查看账号状态和操作记录。

本系统希望通过统一后台管理多个微博账号，实现账号登录、账号状态查看、分组管理、内容编辑、单账号发帖、多账号批量发帖，以及发帖成功后的自动评论功能，提升账号运营效率。

---

## 3. 产品目标

### 3.1 核心目标

1. 支持添加并登录多个微博账号。
2. 每个微博账号拥有独立登录状态。
3. 支持在系统右侧像浏览器一样操作对应微博账号。
4. 支持单个微博账号发帖。
5. 支持多个微博账号同时发帖。
6. 支持发帖成功后自动评论。
7. 支持账号分组管理。
8. 支持查看账号、发帖、评论等状态日志。

### 3.2 合规目标

1. 仅支持用户本人或已获得授权的微博账号。
2. 登录过程由用户在微博官方页面中人工完成。
3. 不保存微博账号明文密码。
4. 不绕过验证码、安全验证或平台风控。
5. 不用于垃圾营销、批量刷屏、恶意评论或其他违反平台规则的行为。
6. 发帖和评论任务必须由用户主动创建或确认。

---

## 4. 产品边界说明

### 4.1 系统支持

1. 用户手动登录微博账号。
2. 管理用户本人或已授权的微博账号。
3. 使用已登录账号进行内容发布。
4. 在用户主动设置的情况下执行发帖后自动评论。
5. 记录登录、发帖、评论、异常等操作日志。
6. 支持账号分组和批量选择。

### 4.2 系统不支持

1. 绕过微博验证码。
2. 破解登录。
3. 自动识别或绕过安全验证。
4. 非授权账号操作。
5. 批量垃圾评论。
6. 恶意刷屏。
7. 规避微博平台风控规则。
8. 违反微博平台规则的自动化行为。

## 9. 账号模块

### 9.1 模块目标

账号模块用于添加微博账号、登录微博账号、展示微博账号头像和名称，并在右侧显示对应账号的微博页面。

### 9.2 页面布局

账号页面分为两部分：

| 区域 | 说明 |
|---|---|
| 左侧账号管理区 | 添加账号、账号列表、账号状态、账号操作 |
| 右侧微博浏览器区 | 展示当前选中账号的微博页面 |

### 9.3 添加账号

用户点击“添加账号”按钮后，系统弹出微博登录页面。

登录流程：

```text
点击添加账号
   ↓
弹出微博登录页面
   ↓
用户手动完成微博登录
   ↓
系统检测登录成功
   ↓
获取微博头像、昵称、UID
   ↓
关闭登录弹窗
   ↓
账号出现在左侧账号列表
   ↓
右侧显示该账号微博页面
```

### 9.4 登录方式

系统采用人工登录方式。

用户在微博官方登录页面中完成：

1. 输入账号密码。
2. 扫码登录。
3. 手机验证码验证。
4. 安全验证。
5. 二次确认。

系统只负责打开登录页面和保存登录成功后的会话状态，不处理验证码破解、安全验证绕过等行为。

### 9.5 登录成功后的展示

登录成功后，账号列表展示：

| 字段 | 说明 |
|---|---|
| 微博头像 | 当前微博账号头像 |
| 微博昵称 | 当前微博账号名称 |
| UID | 微博账号唯一标识 |
| 登录状态 | 在线、离线、登录失效、异常 |
| 所属分组 | 当前账号所属分组 |
| 最近登录时间 | 最近一次成功登录时间 |

示例：

```text
账号列表
--------------------------------
[头像] 账号A    在线     品牌号组
[头像] 账号B    离线     测试号组
[头像] 账号C    登录失效  活动号组
--------------------------------
```

### 9.6 账号状态

| 状态 | 说明 |
|---|---|
| 未登录 | 账号已添加，但未完成登录 |
| 登录中 | 正在打开微博登录页面 |
| 在线 | 当前账号登录正常 |
| 离线 | 当前账号未保持登录 |
| 登录失效 | 登录态失效，需要重新登录 |
| 异常 | 页面加载失败、网络异常或账号状态异常 |
| 发帖中 | 当前账号正在执行发帖任务 |
| 评论中 | 当前账号正在执行自动评论任务 |

### 9.7 右侧微博浏览器区

右侧区域用于展示当前账号对应的微博页面，使用体验接近浏览器。

功能要求：

1. 显示当前选中的微博账号页面。
2. 顶部标题显示当前微博昵称。
3. 支持账号之间切换。
4. 每个账号拥有独立浏览器会话。
5. 账号之间 Cookie、LocalStorage、SessionStorage 互不影响。
6. 用户可以在右侧手动操作微博页面。

示例：

```text
-------------------------------------------------
| 当前账号：账号A                                  |
|------------------------------------------------|
|                                                |
|              微博网页内容区域                   |
|                                                |
-------------------------------------------------
```

## 10. 发帖模块

### 10.1 模块目标

发帖模块用于创建微博内容，并选择一个或多个微博账号进行发布。

核心能力包括：

1. 单账号发帖。
2. 多账号同时发帖。
3. 支持话题。
4. 支持图片。
5. 支持发帖后自动评论。
6. 支持查看每个账号的发帖结果。

### 10.2 发帖页面结构

```text
发帖页面
├── 账号选择区
├── 内容编辑区
├── 话题提示区
├── 图片上传区
├── 自动评论设置区
├── 发布操作区
└── 发布结果区
```

### 10.3 账号选择区

用户可以选择一个或多个账号进行发帖。

支持：

1. 选择单个账号。
2. 选择多个账号。
3. 按分组选择账号。
4. 全选某个分组下的账号。
5. 仅显示在线账号。
6. 自动过滤登录失效账号。
7. 展示账号当前状态。

示例：

```text
选择发帖账号：

[ ] 账号A    在线
[ ] 账号B    在线
[ ] 账号C    登录失效
[ ] 账号D    在线

分组选择：
[品牌号组] [测试号组] [活动号组]
```

### 10.4 内容编辑区

用户可以在系统内编辑微博内容，体验尽量接近微博官方发帖框。

| 字段 | 是否必填 | 说明 |
|---|---|---|
| 微博正文 | 是 | 需要发布的微博文字内容 |
| 话题 | 否 | 支持 `#话题#` 格式 |
| 图片 | 否 | 支持上传本地图片 |
| 发布账号 | 是 | 至少选择一个账号 |
| 是否自动评论 | 否 | 开启后发帖成功自动评论 |
| 评论内容 | 开启自动评论时必填 | 自动评论的内容 |

### 10.5 微博正文要求

微博正文输入框支持：

1. 文本输入。
2. 换行。
3. Emoji。
4. `#话题#`。
5. `@用户`。
6. 字数统计。
7. 空内容校验。
8. 超出字数提醒。

示例：

```text
今天的内容更新来了！#微博运营# #内容发布#
```

### 10.6 话题提示

当用户输入 `#` 时，系统展示话题提示。

话题来源可以包括：

1. 本地话题库。
2. 用户历史使用话题。
3. 手动维护的话题列表。
4. 微博页面自身的话题联想能力。

示例：

```text
输入：#微博

提示：
#微博运营#
#微博营销#
#微博热搜#
#微博内容创作#
```

### 10.7 图片上传

用户可以为微博添加图片。

支持：

1. 上传一张或多张图片。
2. 图片预览。
3. 删除已上传图片。
4. 图片格式校验。
5. 图片大小校验。
6. 上传失败提示。

| 格式 | 是否支持 |
|---|---|
| JPG | 支持 |
| PNG | 支持 |
| WEBP | 支持 |
| GIF | 可后续支持 |

## 11. 单账号发帖

### 11.1 功能说明

用户选择一个微博账号，填写微博内容后发布。

### 11.2 操作流程

```text
进入发帖页面
   ↓
选择一个微博账号
   ↓
输入微博正文
   ↓
添加话题
   ↓
添加图片，可选
   ↓
设置是否自动评论
   ↓
点击发布
   ↓
系统执行发帖
   ↓
返回发帖结果
   ↓
记录状态日志
```

### 11.3 业务规则

1. 未选择账号时，不能发布。
2. 账号未登录时，不能发布。
3. 微博正文为空时，不能发布。
4. 图片上传失败时，需要提示用户。
5. 发帖成功后，需要记录微博链接或微博 ID。
6. 发帖失败时，需要记录失败原因。
7. 如果开启自动评论，发帖成功后进入自动评论流程。

---

## 12. 多账号发帖

### 12.1 功能说明

用户可以选择多个微博账号，同时发布微博内容。

每个账号独立执行任务，互不影响。

### 12.2 操作流程

```text
进入发帖页面
   ↓
选择多个微博账号
   ↓
输入微博正文
   ↓
添加话题和图片
   ↓
设置是否自动评论
   ↓
点击多账号发布
   ↓
系统创建主任务
   ↓
为每个账号创建子任务
   ↓
逐个或并发执行发帖
   ↓
记录每个账号发帖结果
   ↓
展示整体发布结果
```

### 12.3 多账号任务规则

1. 多账号发帖创建一个主任务。
2. 每个账号生成一个子任务。
3. 某个账号失败，不影响其他账号继续执行。
4. 每个账号单独记录发帖状态。
5. 每个账号单独记录微博链接。
6. 每个账号单独判断是否需要自动评论。
7. 多账号任务需要展示整体进度。
8. 多账号发布前需要展示账号数量和内容预览，并由用户确认。

### 12.4 多账号发帖结果

| 账号 | 发帖状态 | 微博链接 | 自动评论状态 | 失败原因 |
|---|---|---|---|---|
| 账号A | 成功 | 已生成 | 成功 | - |
| 账号B | 成功 | 已生成 | 失败 | 评论提交失败 |
| 账号C | 失败 | - | 未执行 | 登录失效 |
| 账号D | 成功 | 已生成 | 未开启 | - |

## 13. 发帖后自动评论

### 13.1 功能目标

系统支持微博发布成功后，自动使用当前发帖账号对刚刚发布的微博进行评论。

### 13.2 自动评论使用场景

用户发布微博后，希望自动追加一条评论，例如：

```text
欢迎大家评论交流～
```

```text
更多内容持续更新中！
```

```text
感谢关注，欢迎转发～
```

### 13.3 自动评论设置

| 设置项 | 是否必填 | 说明 |
|---|---|---|
| 是否开启自动评论 | 否 | 用户手动开启 |
| 评论内容 | 开启后必填 | 自动评论的文字 |
| 评论延迟时间 | 否 | 发帖成功后等待几秒再评论 |
| 评论失败重试 | 否 | 评论失败后是否重试 |
| 最大重试次数 | 否 | 默认 1 次，避免无限重试 |
| 评论账号 | 是 | 默认使用当前发帖账号 |

### 13.4 自动评论流程

```text
微博发布成功
   ↓
获取微博链接 / 微博 ID
   ↓
判断是否开启自动评论
   ↓
校验评论内容
   ↓
等待评论延迟时间
   ↓
打开微博详情页
   ↓
输入评论内容
   ↓
提交评论
   ↓
判断评论是否成功
   ↓
记录评论结果
```

### 13.5 自动评论规则

1. 只有微博发布成功后，才执行自动评论。
2. 如果微博发布失败，不执行自动评论。
3. 如果无法获取微博链接，不执行自动评论。
4. 如果评论内容为空，不执行自动评论。
5. 如果账号登录失效，不执行自动评论。
6. 如果微博页面出现安全验证，暂停任务并提示用户手动处理。
7. 评论失败后可以按配置重试。
8. 重试次数不能无限增加。
9. 评论结果必须写入状态日志。
10. 自动评论仅用于当前账号对自己刚发布内容的补充说明，不用于刷屏或骚扰其他用户。

## 14. 分组模块

### 14.1 模块目标

分组模块用于对多个微博账号进行分类管理，方便用户批量选择账号进行发帖。

### 14.2 分组功能

支持：

1. 新建分组。
2. 修改分组名称。
3. 删除分组。
4. 添加账号到分组。
5. 从分组移除账号。
6. 按分组查看账号。
7. 发帖时按分组选择账号。
8. 统计每个分组下的账号数量。

### 14.3 分组字段

| 字段 | 说明 |
|---|---|
| 分组名称 | 用户自定义名称 |
| 账号数量 | 当前分组下账号数量 |
| 创建时间 | 分组创建时间 |
| 更新时间 | 分组最后更新时间 |
| 备注 | 分组说明 |

### 14.4 分组示例

```text
品牌号组
├── 账号A
├── 账号B
└── 账号C

活动号组
├── 账号D
├── 账号E
└── 账号F
```

## 15. 状态日志模块

### 15.1 模块目标

状态日志模块用于记录系统中所有重要操作，方便用户查看执行情况和排查问题。

### 15.2 日志类型

| 日志类型 | 说明 |
|---|---|
| 登录日志 | 账号登录成功、失败、失效 |
| 发帖日志 | 发帖任务创建、执行、成功、失败 |
| 评论日志 | 自动评论执行、成功、失败 |
| 分组日志 | 分组新增、修改、删除 |
| 系统日志 | 系统异常、页面异常、网络异常 |

### 15.3 日志字段

## 16. 核心业务流程

### 16.1 添加账号流程

```text
用户点击添加账号
   ↓
系统打开微博登录页面
   ↓
用户人工登录
   ↓
系统检测登录状态
   ↓
获取账号头像、昵称、UID
   ↓
保存账号信息
   ↓
关闭登录弹窗
   ↓
账号进入账号列表
```

### 16.2 切换账号流程

```text
用户点击账号列表中的某个账号
   ↓
系统切换当前账号
   ↓
右侧浏览器区域加载该账号微博页面
   ↓
顶部标题显示当前微博昵称
```

### 16.3 单账号发帖 + 自动评论流程

```text
选择一个账号
   ↓
输入微博内容
   ↓
添加图片和话题
   ↓
开启自动评论
   ↓
填写评论内容
   ↓
点击发布
   ↓
系统发布微博
   ↓
发布成功后获取微博链接
   ↓
打开微博详情页
   ↓
发表评论
   ↓
记录发帖和评论结果
```

### 16.4 多账号发帖 + 自动评论流程

```text
选择多个账号
   ↓
输入微博内容
   ↓
添加图片和话题
   ↓
开启自动评论
   ↓
填写评论内容
   ↓
点击多账号发布
   ↓
系统创建主任务
   ↓
为每个账号创建子任务
   ↓
账号逐个执行发帖
   ↓
每个账号发帖成功后执行评论
   ↓
记录每个账号的发帖和评论结果
   ↓
展示整体任务结果
```

## 18. 非功能需求

### 18.1 性能需求

| 项目 | 要求 |
|---|---|
| 页面响应 | 普通操作响应时间小于 2 秒 |
| 账号切换 | 切换账号后页面应快速加载 |
| 多账号发帖 | 支持多个账号任务独立执行 |
| 日志查询 | 支持按时间、账号、状态快速查询 |

### 18.2 安全需求

1. 不保存微博账号明文密码。
2. 登录操作在微博官方页面中完成。
3. Cookie、Token 等登录信息需要加密存储。
4. 不同账号之间登录信息隔离。
5. 支持手动删除账号登录信息。
6. 支持登录失效后重新登录。
7. 发帖和评论前需要由用户主动创建任务。
8. 不允许绕过验证码或安全验证。
9. 多账号发帖需要频率限制和人工确认，避免误操作。

### 18.3 稳定性需求

1. 某个账号任务失败，不影响其他账号任务。
2. 页面加载失败时，需要提示用户。
3. 发帖失败时，需要记录失败原因。
4. 评论失败时，需要记录失败原因。
5. 系统异常不能导致全部任务中断。
6. 任务执行状态需要实时更新。

## 21. 验收标准

### 21.1 账号模块验收标准

1. 用户点击添加账号后，可以正常打开微博登录页面。
2. 用户手动登录成功后，系统可以识别微博头像和昵称。
3. 登录成功后，账号出现在左侧账号列表。
4. 点击不同账号，右侧微博页面可以正确切换。
5. 多个账号之间登录状态互不影响。
6. 登录失效账号可以被标记为异常或失效状态。

### 21.2 发帖模块验收标准

1. 用户可以选择一个账号发帖。
2. 用户可以选择多个账号发帖。
3. 未选择账号时不能发帖。
4. 微博正文为空时不能发帖。
5. 发帖成功后可以记录微博链接。
6. 发帖失败后可以记录失败原因。
7. 多账号发帖时，一个账号失败不影响其他账号。
8. 多账号发帖前必须展示发布账号和内容预览，由用户确认。

### 21.3 自动评论验收标准

1. 用户可以开启或关闭发帖后自动评论。
2. 开启自动评论时，评论内容不能为空。
3. 微博发帖成功后，系统可以进入评论流程。
4. 评论成功后记录评论状态。
5. 评论失败后记录失败原因。
6. 微博发布失败时，不执行自动评论。
7. 无法获取微博链接时，不执行自动评论。
8. 出现验证码或安全验证时，暂停任务并提示用户手动处理。

### 21.4 分组模块验收标准

1. 用户可以创建分组。
2. 用户可以编辑分组名称。
3. 用户可以删除分组。
4. 用户可以将账号加入分组。
5. 发帖时可以按分组选择账号。

### 21.5 状态日志验收标准

1. 登录成功和失败均有日志。
2. 发帖成功和失败均有日志。
3. 评论成功和失败均有日志。
4. 日志中包含时间、账号、类型、状态和原因。
5. 用户可以根据账号和状态筛选日志。

## 23. 推荐技术方案

### 23.1 前端

可选方案：

1. Electron
2. Vue 3
3. React
4. Tauri

如果需要内嵌微博页面，可考虑桌面端内嵌浏览器能力。

### 23.2 后端

可选方案：

1. Node.js
2. NestJS
3. Python FastAPI
4. Java Spring Boot

### 23.3 账号会话隔离

建议每个账号使用独立浏览器上下文或独立用户数据目录，避免账号 Cookie 混用。

```text
profiles/
├── account_001/
├── account_002/
├── account_003/
```

每个账号独立保存：

1. Cookie
2. LocalStorage
3. SessionStorage
4. 浏览器缓存
5. 登录状态

### 23.4 数据库

可选方案：

1. SQLite，适合本地桌面端。
2. MySQL，适合服务端部署。
3. PostgreSQL，适合复杂数据管理。

MVP 阶段建议：

```text
Electron + Vue 3 + SQLite
```

微博发帖、评论能力优先采用微博官方允许的接口或合规页面能力，避免规避平台规则。

## 24. 风险与注意事项

| 风险 | 说明 | 处理方式 |

| 登录失效 | 微博登录态可能过期 | 提醒用户重新登录 |

本产品的核心是帮助用户统一管理多个微博账号，并在合法授权的前提下提高内容发布效率。

第一版重点应优先完成：

```text
微博账号登录
   ↓
多账号独立管理
   ↓
右侧微博页面操作
   ↓
单账号发帖
   ↓
多账号发帖
   ↓
发帖后自动评论
   ↓
状态日志记录
```

核心功能优先级为：

```text
登录微博账号 > 多账号管理 > 发帖 > 多账号发帖 > 自动评论 > 分组 > 日志
```
